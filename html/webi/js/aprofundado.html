Conceito Importante do JS

Closures & Escopo Léxico

Uma closure é uma função que lembra o escopo em que foi criada, mesmo depois que esse escopo deixou de existir. É o mecanismo por trás de módulos, dados privados, callbacks e muito mais.

Escopo Léxico
Em JS o escopo é determinado pelo local onde a função é definida no código, não onde é chamada. Funções internas sempre enxergam as variáveis do escopo externo.
Closure
Quando uma função interna é retornada ou passada como argumento, ela carrega consigo uma referência viva ao escopo onde nasceu — incluindo todas as variáveis daquele escopo.
Variável "Capturada"
A função interna não copia o valor — ela guarda uma referência. Se a variável mudar depois, a closure verá o novo valor.
Dado Privado
Closures permitem criar variáveis inacessíveis de fora — o padrão clássico de encapsulamento antes das classes.
closures.js 
// ── Closure básica: contador privado ──────────
function criarContador() {
  let contagem = 0;  // variável privada — ninguém de fora acessa

  return {
    incrementar() { contagem++; },
    decrementar() { contagem--; },
    valor()       { return contagem; }
  };
}

const c1 = criarContador();
const c2 = criarContador();  // escopo completamente independente!

c1.incrementar();
c1.incrementar();
console.log(c1.valor());  // 2
console.log(c2.valor());  // 0 — c2 tem sua própria contagem

// ── Fábrica de funções ────────────────────────
function multiplicadorDe(fator) {
  return n => n * fator;  // fator é capturado pela closure
}

const dobrar    = multiplicadorDe(2);
const quadruplicar = multiplicadorDe(4);
console.log(dobrar(5));       // 10
console.log(quadruplicar(5)); // 20

// ── Memoização com closure ────────────────────
function memoizar(fn) {
  const cache = new Map();
  return function(...args) {
    const chave = JSON.stringify(args);
    if (cache.has(chave)) return cache.get(chave);
    const resultado = fn.apply(this, args);
    cache.set(chave, resultado);
    return resultado;
  };
}

const fibMemo = memoizar((n) =>
  n <= 1 ? n : fibMemo(n - 1) + fibMemo(n - 2)
);
console.log(fibMemo(40));  // rápido! resultado em cache
⚠️
O bug clássico do var em loops: Ao usar var dentro de um for com callbacks, todas as closures compartilham a mesma variável. Com let isso não acontece — cada iteração tem seu próprio escopo.
❌ Bug com var
for (var i = 0; i < 3; i++) {
  setTimeout(() =>
    console.log(i)  // 3, 3, 3
  , 100);
}
// todas as closures capturam
// o mesmo "i", que já é 3
✅ Correto com let
for (let i = 0; i < 3; i++) {
  setTimeout(() =>
    console.log(i)  // 0, 1, 2
  , 100);
}
// let cria um "i" novo
// em cada iteração do bloco
Herança em JavaScript

Cadeia de Protótipos

JavaScript não tem herança clássica — tem herança baseada em protótipos. Cada objeto tem um link oculto [[Prototype]] que aponta para outro objeto. A busca por propriedades sobe essa cadeia automaticamente.

[[Prototype]]
Link interno de cada objeto. Acessível via Object.getPrototypeOf(obj) ou obj.__proto__ (legado).
Busca Prototípica
Ao acessar uma propriedade, o JS procura no próprio objeto, depois no protótipo, depois no protótipo do protótipo… até chegar em null.
prototype (fn)
Toda função tem uma propriedade .prototype. Quando usada com new, o objeto criado tem seu [[Prototype]] apontando para ela.
Object.create()
Cria um novo objeto cujo [[Prototype]] é o argumento passado. A forma mais direta de herança prototípica.

prototipos.js 
// ── Object.create: herança explícita ─────────
const animal = {
  respirar() { console.log(`${this.nome} respira.`); }
};

const cachorro = Object.create(animal);
cachorro.nome = "Rex";
cachorro.latir = () => console.log("Au au!");

cachorro.latir();     // "Au au!" — próprio
cachorro.respirar(); // "Rex respira." — herdado!

console.log(Object.getPrototypeOf(cachorro) === animal); // true

// ── hasOwnProperty: minha ou herdada? ────────
console.log(cachorro.hasOwnProperty("nome"));    // true
console.log(cachorro.hasOwnProperty("respirar")); // false

// ── Adicionando ao prototype retroativamente ─
Array.prototype.soma = function() {
  return this.reduce((a, b) => a + b, 0);
};
console.log([1, 2, 3, 4].soma());  // 10
// ⚠️ Nunca faça isso em produção — polui o prototype global
OOP Moderno

Classes Avançadas

A sintaxe class do ES6 é açúcar sintático sobre protótipos — mas adiciona campos privados, métodos estáticos, getters/setters e herança muito mais limpa.

#campo Privado
Campos com # são verdadeiramente privados — inacessíveis fora da classe, até mesmo no console.
static
Métodos/campos da classe, não das instâncias. Chamados como Classe.metodo(), sem new.
get / set
Accessors que parecem propriedades mas executam lógica. obj.nome pode chamar uma função.
extends + super
extends herda de outra classe. super() chama o constructor pai. super.metodo() chama método do pai.
classes-avancadas.js 
class ContaBancaria {
  // Campo privado — não acessível de fora
  #saldo = 0;
  #historico = [];

  // Campo estático
  static taxaJuros = 0.05;

  constructor(public titular, saldoInicial = 0) {
    this.titular = titular;
    this.#depositar(saldoInicial);
  }

  // Getter — acessa como propriedade
  get saldo() { return this.#saldo; }

  // Método privado
  #registrar(operacao, valor) {
    this.#historico.push({ operacao, valor, data: new Date() });
  }

  #depositar(valor) {
    if (valor <= 0) throw new Error("Valor inválido");
    this.#saldo += valor;
    this.#registrar("depósito", valor);
  }

  depositar(valor) { this.#depositar(valor); return this; }

  sacar(valor) {
    if (valor > this.#saldo) throw new Error("Saldo insuficiente");
    this.#saldo -= valor;
    this.#registrar("saque", valor);
    return this;  // permite encadeamento
  }

  extrato() { return [...this.#historico]; }

  // Método estático — usado sem instância
  static calcularRendimento(valor, meses) {
    return valor * (1 + ContaBancaria.taxaJuros) ** meses;
  }
}

// ── Herança ───────────────────────────────────
class ContaPoupanca extends ContaBancaria {
  #rendimentoAcumulado = 0;

  constructor(titular, saldoInicial) {
    super(titular, saldoInicial);  // obrigatório antes de "this"
  }

  renderMensal() {
    const rendimento = this.saldo * ContaBancaria.taxaJuros;
    this.depositar(rendimento);
    this.#rendimentoAcumulado += rendimento;
    return rendimento;
  }
}

const conta = new ContaBancaria("Ana", 1000);
conta.depositar(500).sacar(200);  // encadeamento
console.log(conta.saldo);           // 1300
console.log(ContaBancaria.calcularRendimento(1000, 12));

A Engine por Baixo

Event Loop & Filas de Tarefas

JavaScript é single-threaded — executa uma coisa por vez. O Event Loop é o mecanismo que permite lidar com operações assíncronas sem travar a thread principal.

Call Stack
Pilha de execução — onde as funções são empilhadas e desempilhadas. JavaScript executa o que está no topo.
Microtask Queue
Fila de alta prioridade: callbacks de Promises (.then) e queueMicrotask(). Esvaziada completamente antes de cada macrotarefa.
Macrotask Queue
Fila comum: setTimeout, setInterval, eventos de I/O. Executa uma tarefa por vez, depois o Event Loop verifica microtasks.
Event Loop
Loop infinito que verifica: a Call Stack está vazia? Então pega a próxima microtask. Microtask queue vazia? Pega a próxima macrotask.

console.log("C")
setTimeout cb
main()
⚡ Microtask Queue

queueMicrotask
— vazia quando stack limpa —
⏱ Macrotask Queue
setTimeout(fn, 0)
setInterval cb

event-loop.js 
console.log("A");                          // 1º — síncrono

setTimeout(() => console.log("B"), 0);     // 4º — macrotask (mesmo com 0ms!)

Promise.resolve()
  .then(() => console.log("C"));           // 3º — microtask

console.log("D");                          // 2º — síncrono

// Ordem de saída: A → D → C → B

// ── Por que microtask antes de macrotask? ─────
async function exemplo() {
  console.log("1 - início async");

  await Promise.resolve();             // pausa aqui → vai para microtask queue

  console.log("3 - depois do await");  // retoma da microtask queue
}

exemplo();
console.log("2 - síncrono continua");

// Saída: 1 → 2 → 3

// ── queueMicrotask: agendar explicitamente ─────
queueMicrotask(() => console.log("microtask manual"));
console.log("síncrono");
// Saída: "síncrono" → "microtask manual"
⚠️
Starvation de macrotasks: Se você criar microtasks dentro de microtasks infinitamente, o Event Loop nunca chegará nas macrotasks — a página "trava" mesmo sem usar a CPU ao máximo.
Organização de Código

Módulos ES (ESM)

Módulos ES são a forma nativa de dividir código JS em arquivos. Cada arquivo é um módulo com seu próprio escopo. Use export para expor e import para consumir.

export nomeado
Exporta múltiplos valores. Importado com { chaves } correspondendo exatamente ao nome.
export default
Uma exportação principal por arquivo. Importada com qualquer nome, sem chaves.
import dinâmico
import() carrega um módulo assincronamente — útil para code splitting e carregamento sob demanda.
type="module"
Atributo HTML necessário para usar ESM no browser. Módulos têm defer implícito e escopo isolado.

export const PI

📊 calculos.js

Iterators & Generators

Generators são funções que podem pausar e retomar execução. Permitem criar sequências de dados sob demanda — ideais para fluxos infinitos, paginação e código assíncrono mais legível.

Iterator Protocol
Qualquer objeto com método next() que retorna { value, done } é um iterator. É o que for...of consome por baixo.
function*
Declara um generator. Retorna um objeto iterador. A função não executa até você chamar .next().
yield
Pausa o generator e entrega um valor. Na próxima chamada de .next(), a execução retoma exatamente de onde parou.
yield*
Delega para outro iterable ou generator, "planificando" sequências compostas.
function* contador(inicio = 0)
function* contador(inicio = 0) {
  let i = inicio;
  while (true) {
    const reset = yield i;
    i = reset ?? i + 1;
  }
}
Execução passo a passo
.next()→ { value: 0, done: false }
.next()→ { value: 1, done: false }
.next(10)→ { value: 10, done: false }
.next()→ { value: 11, done: false }
.return()→ { value: undef, done: true }
generators.js 
// ── Sequência infinita com lazy evaluation ────
function* fibonacci() {
  let [a, b] = [0, 1];
  while (true) {
    yield a;
    [a, b] = [b, a + b];
  }
}

// Pegar os primeiros 8 fibonacci
function take(gen, n) {
  const resultado = [];
  for (const val of gen) {
    resultado.push(val);
    if (resultado.length === n) break;
  }
  return resultado;
}
console.log(take(fibonacci(), 8));
// [0, 1, 1, 2, 3, 5, 8, 13]

// ── Generator para paginação de API ───────────
async function* paginarAPI(url) {
  let pagina = 1;
  while (true) {
    const res  = await fetch(`${url}?page=${pagina}`);
    const dados = await res.json();
    if (dados.length === 0) return;  // sem mais dados
    yield dados;
    pagina++;
  }
}

// for await...of consome async generators
for await (const lote of paginarAPI("/api/posts")) {
  console.log(`Lote com ${lote.length} posts`);
}
Metaprogramação

Proxy & Reflect

Proxy permite interceptar e redefinir operações fundamentais em objetos — leitura, escrita, deleção, chamada de função. É a base de sistemas de reatividade como o Vue 3.

new Proxy(alvo, handler)
Cria um proxy em volta do objeto alvo. O handler define quais operações serão interceptadas (as "armadilhas" — traps).
Reflect
API que espelha as operações do Proxy. Permite executar o comportamento padrão dentro de uma trap, como Reflect.get(alvo, prop).
get / set traps
As mais usadas. Interceptam leitura e escrita de propriedades — base para validação, observabilidade e computed props.
Reatividade
O Vue 3 usa Proxy para detectar quando dados mudam e re-renderizar automaticamente o DOM — tudo via trap set.
get
Intercepta leitura de propriedade.
obj.nome → trap
set
Intercepta atribuição de valor.
obj.nome = "x" → trap
has
Intercepta operador in.
"nome" in obj → trap
deleteProperty
Intercepta delete.
delete obj.x → trap
apply
Intercepta chamada de função.
fn() → trap
construct
Intercepta new.
new Classe() → trap
ownKeys
Intercepta Object.keys().
Object.keys(obj) → trap
defineProperty
Intercepta Object.defineProperty.
Metadados customizados
proxy.js 
// ── Validação automática de schema ────────────
function criarValidado(alvo, schema) {
  return new Proxy(alvo, {
    set(obj, prop, valor) {
      const regra = schema[prop];
      if (regra && typeof valor !== regra) {
        throw new TypeError(
          `${prop} deve ser ${regra}, recebeu ${typeof valor}`
        );
      }
      return Reflect.set(obj, prop, valor);  // comportamento normal
    }
  });
}

const usuario = criarValidado({}, {
  nome: "string",
  idade: "number"
});

usuario.nome  = "Ana";   // ✅ ok
usuario.idade = 20;     // ✅ ok
usuario.idade = "vinte"; // ❌ TypeError: idade deve ser number

// ── Objeto reativo (mini Vue) ─────────────────
function reativo(dado, callback) {
  return new Proxy(dado, {
    set(obj, prop, valor) {
      const anterior = obj[prop];
      Reflect.set(obj, prop, valor);
      if (anterior !== valor) callback(prop, valor, anterior);
      return true;
    }
  });
}

const estado = reativo(
  { contador: 0 },
  (prop, novo, velho) =>
    console.log(`${prop}: ${velho} → ${novo}`)
);
estado.contador++;  // "contador: 0 → 1"

Soluções Reutilizáveis

Design Patterns em JS

Padrões de projeto são soluções testadas para problemas recorrentes. Não são regras rígidas — são vocabulário e guias que tornam código mais legível, reutilizável e testável.

🔒 Singleton Criacional
Garante que uma classe tenha apenas uma instância e fornece um ponto global de acesso. Útil para configurações, conexões DB, loggers.

getInstance() retorna a mesma instância sempre.
🏭 Factory Criacional
Encapsula a lógica de criação de objetos. Em vez de new direto, uma função decide qual classe criar com base em parâmetros.

Desacopla o código de criação do código de uso.
👁️ Observer Comportamental
Define uma dependência um-para-muitos: quando um objeto muda, todos os seus observadores são notificados. Base de eventos, state management e reatividade.

subscribe(fn) / notify(dado)
🔌 Strategy Comportamental
Define uma família de algoritmos, encapsula cada um e os torna intercambiáveis. Troca o comportamento de um objeto em runtime sem alterar sua estrutura.

Ex: diferentes estratégias de ordenação ou pagamento.
🎭 Decorator Estrutural
Adiciona comportamento extra a um objeto ou função sem alterar sua implementação original. Muito comum em frameworks (middlewares, HOCs no React).

Ex: adicionar cache, log ou validação a uma função.
🚇 Mediator Comportamental
Define um objeto que coordena a comunicação entre outros objetos, evitando dependências diretas. Componentes falam com o mediator, não entre si.

Ex: EventBus, Redux store.
patterns.js 
// ── Observer Pattern ──────────────────────────
class EventEmitter {
  #listeners = new Map();

  on(evento, callback) {
    if (!this.#listeners.has(evento)) {
      this.#listeners.set(evento, new Set());
    }
    this.#listeners.get(evento).add(callback);
    return () => this.off(evento, callback);  // retorna unsubscribe
  }

  off(evento, callback) {
    this.#listeners.get(evento)?.delete(callback);
  }

  emit(evento, ...args) {
    this.#listeners.get(evento)?.forEach(cb => cb(...args));
  }

  once(evento, callback) {
    const wrapper = (...args) => {
      callback(...args);
      this.off(evento, wrapper);
    };
    this.on(evento, wrapper);
  }
}

// Uso
const bus = new EventEmitter();
const parar = bus.on("login", (user) => console.log(`Bem-vindo, ${user}!`));
bus.emit("login", "Ana");  // "Bem-vindo, Ana!"
parar();                   // remove listener
bus.emit("login", "Bruno"); // silêncio

// ── Factory Pattern ───────────────────────────
function criarNotificacao(tipo, mensagem) {
  const tipos = {
    sucesso: (msg) => ({ icone: "✅", cor: "verde", msg }),
    erro:    (msg) => ({ icone: "❌", cor: "vermelho", msg }),
    aviso:   (msg) => ({ icone: "⚠️", cor: "amarelo", msg }),
  };
  const fabrica = tipos[tipo] ?? tipos.aviso;
  return fabrica(mensagem);
}
console.log(criarNotificacao("sucesso", "Salvo!"));
O Browser como Plataforma
APIs Nativas do Browser

Web APIs Modernas

O browser expõe APIs poderosas que vão muito além do DOM básico. Estas são as mais usadas em aplicações modernas — e as mais cobradas em entrevistas.

IntersectionObserver
Detecta quando um elemento entra ou sai da viewport. Lazy loading, animações no scroll e infinite scroll — sem scroll event.
MutationObserver
Observa mudanças no DOM — inserção/remoção de nós, mudança de atributos. Útil para extensões de browser e integrações legadas.
ResizeObserver
Observa mudanças no tamanho de elementos. Muito mais preciso que window.resize para componentes responsivos.
Web Workers
Executa JS em uma thread separada. Útil para cálculos pesados que travariam a UI. Comunicação via postMessage.
AbortController
Cancela operações assíncronas em andamento — como um fetch() que levou muito tempo ou que o usuário desistiu.
structuredClone()
Faz cópia profunda de objetos nativamente, sem JSON.parse/stringify. Suporta Date, Map, Set, ArrayBuffer.

// ── AbortController: cancelar fetch ──────────
async function buscarComTimeout(url, ms = 5000) {
  const controller = new AbortController();
  const timeout = setTimeout(
    () => controller.abort(), ms
  );

  try {
    const res = await fetch(url, {
      signal: controller.signal
    });
    clearTimeout(timeout);
    return await res.json();
  } catch (e) {
    if (e.name === "AbortError") {
      throw new Error("Timeout: requisição cancelada");
    }
    throw e;
  }
}

🚀
Promise.all, race, allSettled, any: Para rodar várias Promises em paralelo, use Promise.all([p1, p2, p3]) (falha se qualquer uma falhar), Promise.allSettled() (espera todas, sucesso ou falha), Promise.race() (retorna a mais rápida) ou Promise.any() (retorna a primeira bem-sucedida).

Desafios Avançados

Exercícios

Cada exercício cobre um tema deste guia. Tente implementar do zero antes de consultar a dica.

EX 01 Pipeline com Closures
  • Crie uma função pipeline(...fns) que recebe funções e retorna uma nova função.
  • A nova função aplica cada função da lista em sequência, passando o resultado para a próxima.
  • Ex: pipeline(dobrar, somar1, quadrado)(3) → ((3*2)+1)² = 49
  • Use reduce internamente para encadear as funções.
💡 Dica: return fns.reduce((acc, fn) => fn(acc), valor) — o acumulador começa sendo o valor inicial e vai sendo transformado por cada função.
EX 02 Gerenciador de Estado com Observer
  • Crie uma classe Store com #estado privado.
  • Implemente subscribe(fn) para registrar listeners, getState() e setState(novoEstado).
  • Sempre que o estado mudar, notifique todos os subscribers com o estado novo e o anterior.
  • Use structuredClone() para garantir que o estado retornado seja uma cópia imutável.
💡 Dica: armazene os listeners em um Set para evitar duplicatas. O setter deve fazer merge: this.#estado = { ...this.#estado, ...novoEstado }.
EX 03 Proxy de Cache Automático
  • Crie uma função comCache(obj) que envolve qualquer objeto num Proxy.
  • Quando um método for chamado, verifique se o resultado já está em cache (use um Map com chave baseada em método + args).
  • Se estiver, retorne do cache. Se não, execute e salve.
  • Teste com um objeto que simula uma operação lenta (use um loop pesado ou setTimeout).
💡 Dica: intercepte a trap get — verifique se o valor é uma função, e se for, retorne uma versão "memoizada" dela usando uma closure que mantém o Map de cache.
EX 04 Scroll Infinito com Generator + IntersectionObserver
  • Crie um generator function* gerarItens() que produz objetos { id, titulo } indefinidamente.
  • Use um IntersectionObserver para detectar quando um sentinela no final da lista fica visível.
  • Ao detectar, chame gen.next() várias vezes e adicione os itens ao DOM com createElement.
  • O sentinela deve sempre ser o último elemento — reposicione-o após adicionar itens.
💡 Dica: o sentinela é um div vazio no final da lista. Quando ele intersectar a viewport, chame o generator 10 vezes em um loop for e faça append de cada item antes do sentinela.
EX 05 Módulo de Autenticação com Singleton + Closure
  • Crie um módulo ESM auth.js com uma função criarAuth() que retorna um singleton.
  • O estado interno (#usuario, #token) deve ser privado.
  • Métodos: login(email, senha) (simule com fetch), logout(), getUsuario(), estaAutenticado().
  • Importe e use o módulo em main.js. O estado deve persistir entre páginas via localStorage.
💡 Dica: no topo do módulo, crie let instancia = null. Em criarAuth(), se instancia já existe, retorne ela — senão crie. Isso é o padrão Singleton.
DESAFIO Mini Framework Reativo
  • Combine Proxy (reatividade), Observer (notificação) e template literals (renderização) para criar um mini framework.
  • A função criar({ estado, template, onMudar }) deve montar um componente num elemento do DOM.
  • Sempre que o estado mudar via Proxy, o DOM deve ser re-renderizado automaticamente.
  • Bônus: implemente computed — valores derivados que recalculam quando o estado muda.
💡 Dica: o template é uma função que recebe o estado e retorna uma string HTML: template: (s) => `<h1>${s.contador}</h1>`. Use innerHTML para atualizar. Não esqueça de reconectar os eventos a cada renderização (ou use event delegation).
