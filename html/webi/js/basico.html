Armazenando Informação

Variáveis & Tipos

Uma variável é um espaço com nome na memória do computador. Em JavaScript, você tem três formas de criar: const, let e (o antigo) var.

const
Valor constante — não pode ser reatribuído. Use sempre que o valor não vai mudar.
let
Variável com escopo de bloco. Pode ser reatribuída. Prefira ao var.
var
A forma antiga. Tem escopo de função e sofre hoisting. Evite em código moderno.
Tipo Primitivo
Valor simples: string, number, boolean, null, undefined.
typeof
Operador que diz qual é o tipo de um valor: typeof 42 → "number".
console.log
Sua melhor ferramenta: imprime valores no console do navegador (F12).
Tipo Exemplo typeof retorna Observação
string "Olá" 'mundo' `texto` "string" Texto entre aspas duplas ou simples
number 42 3.14 -7 "number" Inteiros e decimais no mesmo tipo
boolean true false "boolean" Resultado de comparações
null null "object" ⚠️ Bug histórico do JS — null não é objeto
undefined undefined "undefined" Variável declarada mas sem valor
variaveis.js 
// ── const: valor não muda ──────────────────────
const nome = "Ana";
const anoNascimento = 2005;
const ehEstudante = true;

// ── let: pode ser reatribuído ──────────────────
let pontuacao = 0;
pontuacao = 100;        // ✅ funciona

// ── Template literals (aspas simples) ──────────────
const idade = 2025 - anoNascimento;
const mensagem = `Olá, ${nome}! Você tem ${idade} anos.`;
console.log(mensagem);    // Olá, Ana! Você tem 20 anos.

// ── typeof ─────────────────────────────────────
console.log(typeof nome);         // "string"
console.log(typeof pontuacao);    // "number"
console.log(typeof ehEstudante);  // "boolean"
💡
Regra prática: Use const por padrão. Só troque para let se precisar reatribuir o valor. Nunca use var em código novo.
Cálculos & Comparações

Operadores

Operadores são símbolos que realizam operações sobre valores. Em JavaScript temos aritméticos, relacionais, lógicos e de atribuição.

Aritméticos

+
Adição / Concatenação
5 + 3 → 8
"Oi" + " " + "JS" → "Oi JS"
-
Subtração
10 - 4 → 6
*
Multiplicação
3 * 7 → 21
/
Divisão
10 / 4 → 2.5
%
Módulo (resto)
10 % 3 → 1
par: n % 2 === 0
**
Potência
2 ** 10 → 1024
++
Incremento
x++ → x = x + 1
--
Decremento
x-- → x = x - 1

Relacionais — retornam true ou false

===
Igual (valor e tipo)
3 === "3" → false
!==
Diferente (estrito)
3 !== "3" → true
>
Maior que
5 > 3 → true
<
Menor que
2 < 8 → true
>=
Maior ou igual
5 >= 5 → true
<=
Menor ou igual
3 <= 9 → true
⚠️
== vs === — Em JS sempre use === (comparação estrita). O == faz conversão de tipos silenciosa e causa bugs: 0 == false → true, 0 === false → false.

Lógicos

&&
E lógico (AND)
true && false → false
ambos devem ser true
||
OU lógico (OR)
true || false → true
pelo menos um true
!
NÃO lógico (NOT)
!true → false
??
Nullish Coalescing
null ?? "padrão" → "padrão"
operadores.js 
const a = 10, b = 3;

// Aritméticos
console.log(a + b);   // 13
console.log(a % b);   // 1 (resto de 10 ÷ 3)
console.log(2 ** 8);   // 256

// Relacionais estritos
console.log(3 === "3");  // false — tipos diferentes
console.log(3 ==  "3");  // true  — evite!

// Lógicos
const maiorDeIdade = 18;
const temCNH = true;
console.log(maiorDeIdade >= 18 && temCNH);  // true

// Nullish coalescing
const apelido = null;
console.log(apelido ?? "Sem apelido");  // "Sem apelido"
Decisões

Condicionais

O programa escolhe um caminho baseado em uma condição. Em JS: if / else if / else para condições dinâmicas e switch para valores fixos.

if
"Se" — executa o bloco se a condição for true.
else if
Encadeia condições extras testadas em ordem.
else
"Senão" — executado quando nenhuma condição anterior foi verdadeira.
switch
Compara uma variável com vários valores fixos via case.
Operador ternário
condição ? seTrue : seFalse — if/else em uma linha.
Truthy / Falsy
Valores que JS trata como verdadeiro/falso: 0, "", null, undefined, NaN são falsy.
INÍCIO
nota >= 7 ?
SIM
"Aprovado ✅"
NÃO
nota >= 5 ?
SIM
"Recuperação ⚠️"
NÃO
"Reprovado ❌"
condicionais.js 
// ── if / else if / else ────────────────────────
const nota = 6.5;

if (nota >= 7) {
  console.log("✅ Aprovado!");
} else if (nota >= 5) {
  console.log("⚠️ Recuperação.");
} else {
  console.log("❌ Reprovado.");
}

// ── Operador ternário ──────────────────────────
const status = nota >= 7 ? "Aprovado" : "Reprovado";

// ── switch ─────────────────────────────────────
const dia = "segunda";
switch (dia) {
  case "sábado":
  case "domingo":
    console.log("🎉 Fim de semana!");
    break;
  default:
    console.log("💼 Dia de semana.");
}
⚠️
Não esqueça o break no switch! — Sem break, a execução "cai" nos case seguintes automaticamente (fall-through). Geralmente é um bug.
Repetição

Laços (Loops)

Laços repetem um bloco de código enquanto uma condição for verdadeira. Sem eles, você teria que repetir o mesmo código.

for
Número de repetições conhecido. Tem inicialização, condição e incremento na mesma linha.
while
Repete enquanto a condição for verdadeira. Testa antes — pode nunca executar.
do…while
Executa ao menos uma vez, depois testa a condição.
for…of
Itera sobre os valores de um array ou string diretamente.
break
Sai do laço imediatamente.
continue
Pula para a próxima iteração sem executar o restante do bloco.

Visualizando o for

for (let i = 0; i < 5; i++)
i = 00 < 5 ✓→executa bloco
i = 11 < 5 ✓→executa bloco
i = 22 < 5 ✓→executa bloco
i = 33 < 5 ✓→executa bloco
i = 44 < 5 ✓→executa bloco
i = 55 < 5 ✗ → PARA
loops.js 
// ── for clássico ───────────────────────────────
for (let i = 1; i <= 5; i++) {
  console.log(`Iteração ${i}`);
}

// ── for…of — percorre arrays ───────────────────
const frutas = ["🍎 Maçã", "🍌 Banana", "🍊 Laranja"];
for (const fruta of frutas) {
  console.log(fruta);
}

// ── while — condição desconhecida ──────────────
let tentativas = 0;
while (tentativas < 3) {
  console.log("Tentando...");
  tentativas++;
}

// ── continue e break ───────────────────────────
for (let i = 1; i <= 10; i++) {
  if (i % 2 === 0) continue;  // pula pares
  if (i > 7) break;            // para em 7
  console.log(i);               // 1 3 5 7
}
Trabalhando com Dados
Coleções

Arrays

Um array é uma lista ordenada de valores. Cada item tem um índice começando em 0. Arrays têm métodos poderosos para transformar dados.

Anatomia de um Array

"Ana"
índice 0
"Bruno"
índice 1
"Carlos"
índice 2
"Diana"
índice 3
→ .length = 4
propriedade
.push(item)
Adiciona ao final.
arr.push("novo") → [... "novo"]
.pop()
Remove e retorna o último.
arr.pop() → remove "Diana"
.unshift(item)
Adiciona ao início.
arr.unshift("Zero") → ["Zero", ...]
.shift()
Remove e retorna o primeiro.
arr.shift() → remove "Ana"
.indexOf(val)
Retorna o índice ou -1.
arr.indexOf("Bruno") → 1
.includes(val)
Verifica se o valor existe.
arr.includes("Ana") → true
.forEach(fn)
Executa uma função para cada item. Não retorna array.
arr.forEach(x => console.log(x))
.map(fn)
Transforma cada item. Retorna novo array.
[1,2,3].map(x => x*2) → [2,4,6]
.filter(fn)
Filtra itens que passam no teste.
[1..10].filter(x => x%2===0)
.reduce(fn, ini)
Acumula todos os itens em um valor.
[1,2,3].reduce((a,b)=>a+b, 0) → 6
.find(fn)
Retorna o primeiro item que passa no teste.
arr.find(x => x > 5)
.sort(fn)
Ordena o array (modifica o original!).
arr.sort((a,b) => a-b)
arrays.js 
const notas = [7.5, 4.0, 9.2, 6.8, 3.5];

// map — transforma cada nota
const arredondadas = notas.map(n => Math.round(n));
// [8, 4, 9, 7, 4]

// filter — só aprovados
const aprovados = notas.filter(n => n >= 7);
// [7.5, 9.2]

// reduce — média da turma
const soma = notas.reduce((acc, n) => acc + n, 0);
const media = soma / notas.length;
console.log(`Média: ${media.toFixed(1)}`);  // "Média: 6.2"

// Encadeamento — média dos aprovados
const mediaAprovados = notas
  .filter(n => n >= 7)
  .reduce((a, b) => a + b, 0) / aprovados.length;
console.log(mediaAprovados);  // 8.35
Estruturas de Dados

Objetos

Objetos armazenam dados como pares chave: valor. São a base do JavaScript — quase tudo é um objeto.

Propriedade
Par chave: valor dentro de um objeto.
Método
Função que é propriedade de um objeto. Usa this para acessar o próprio objeto.
Desestruturação
Extrai propriedades em variáveis: const { nome, idade } = pessoa;
Spread (...)
Copia ou mescla objetos: const novo = { ...obj1, ...obj2 };
this
Dentro de um método, this aponta para o próprio objeto.
Optional chaining (?.)
Acessa propriedades sem erro se o objeto for null: user?.address?.city
objetos.js 
// ── Criando um objeto ──────────────────────────
const aluno = {
  nome: "Bruno",
  idade: 17,
  notas: [8, 7.5, 9],
  calcMedia() {
    const soma = this.notas.reduce((a, b) => a + b, 0);
    return soma / this.notas.length;
  }
};

console.log(aluno.nome);          // "Bruno"
console.log(aluno.calcMedia());  // 8.17

// ── Desestruturação ────────────────────────────
const { nome, idade } = aluno;

// ── Spread — copiar e atualizar ────────────────
const alunoAtualizado = { ...aluno, idade: 18 };

// ── Optional chaining ──────────────────────────
const cidade = aluno?.endereco?.cidade ?? "Não informado";
console.log(cidade);  // "Não informado"

// ── Array de objetos ───────────────────────────
const turma = [
  { nome: "Ana", nota: 9 },
  { nome: "Bruno", nota: 6 },
  { nome: "Carla", nota: 8 },
];
const melhores = turma.filter(({ nota }) => nota >= 7);
Reutilizando Código

Funções

Funções são blocos de código reutilizáveis com nome. Em JavaScript, funções são "cidadãos de primeira classe" — podem ser guardadas em variáveis, passadas como argumentos e retornadas por outras funções.

Declaração
function nome() {} — sofre hoisting, pode ser chamada antes de ser declarada.
Expressão
const fn = function() {} — atribuída a uma variável, sem hoisting.
Arrow Function
const fn = () => {} — sintaxe curta do ES6. Não tem seu próprio this.
Callback
Função passada como argumento para outra função e chamada depois.
Parâmetro padrão
function fn(x = 10) {} — valor usado se o argumento for undefined.
return
Encerra a função e devolve um valor. Sem return → retorna undefined.
funcoes.js 
// ── Declaração de função ───────────────────────
function saudar(nome, hora = "dia") {
  return `Bom ${hora}, ${nome}!`;
}
console.log(saudar("Ana"));          // "Bom dia, Ana!"
console.log(saudar("Bruno", "noite")); // "Boa noite, Bruno!"

// ── Arrow function ─────────────────────────────
const dobrar = n => n * 2;  // return implícito
console.log(dobrar(5));       // 10

// ── Callback ───────────────────────────────────
const numeros = [3, 1, 4, 1, 5];
const pares = numeros.filter(n => n % 2 === 0);  // arrow como callback
const dobrados = numeros.map(dobrar);          // função como callback

// ── Funções que retornam funções ───────────────
function multiplicador(fator) {
  return n => n * fator;
}
const triplicar = multiplicador(3);
console.log(triplicar(7));  // 21
🚀
Arrow functions com return implícito: Quando o corpo da função é uma única expressão, pode-se omitir {} e return: const dobrar = n => n * 2 é igual a const dobrar = n => { return n * 2; }.
O DOM
Manipulando a Página

DOM — Document Object Model

O DOM é a representação da página HTML como objetos JS. Com ele você seleciona elementos, muda conteúdo, adiciona classes, cria novos elementos e responde a eventos do usuário.

querySelector
Seleciona o primeiro elemento que bate com o seletor CSS.
querySelectorAll
Retorna todos os elementos como NodeList.
innerHTML / textContent
innerHTML — HTML dentro do elemento. textContent — só texto, mais seguro.
classList
Manipula classes: .add(), .remove(), .toggle(), .contains().
addEventListener
Registra uma função para ser chamada quando um evento ocorre.
createElement
Cria um novo elemento HTML: document.createElement("div").
dom.js 
// ── Selecionar elementos ───────────────────────
const titulo = document.querySelector("h1");
const botao  = document.querySelector("#meu-btn");
const itens  = document.querySelectorAll(".item");

// ── Alterar conteúdo ───────────────────────────
titulo.textContent = "Novo título!";
titulo.style.color = "#f7df1e";

// ── Classes ────────────────────────────────────
titulo.classList.add("destaque");
titulo.classList.toggle("visivel");

// ── Eventos ────────────────────────────────────
botao.addEventListener("click", () => {
  titulo.textContent = "Clicou!";
});

// ── Criar elemento dinamicamente ───────────────
function criarItem(texto) {
  const li = document.createElement("li");
  li.textContent = texto;
  li.classList.add("item");
  document.querySelector("#lista").appendChild(li);
}

// ── Event delegation ───────────────────────────
document.querySelector("#lista").addEventListener("click", (e) => {
  if (e.target.classList.contains("item")) {
    e.target.classList.toggle("concluido");
  }
});
💡
Eventos comuns: click, input, submit, keydown, keyup, mouseover, change, DOMContentLoaded. Use e.preventDefault() para cancelar o comportamento padrão (ex: evitar recarregar a página no submit de um form).
Async & APIs
Código Assíncrono

Async, Promises & Fetch

JavaScript executa de forma não-bloqueante: operações lentas (rede, timer) acontecem "em paralelo". Promises e async/await são a forma moderna de lidar com isso.

setTimeout
Executa uma função após X milissegundos. Base dos callbacks assíncronos.
Promise
Representa um valor que estará disponível no futuro. Pode ser pending, resolved ou rejected.
.then() / .catch()
Encadeia ações após a Promise resolver (then) ou falhar (catch).
async / await
Sintaxe que faz código assíncrono parecer síncrono. Mais legível que .then().
fetch()
API nativa para fazer requisições HTTP. Retorna uma Promise.
try / catch
Captura erros em código async. Sempre use com await fetch().

Como uma Promise funciona

new Promise()
Cria a promessa. Recebe uma função com resolve e reject. O estado inicial é pending.
resolve("dados")
Sucesso! O valor passado vai para o próximo .then().
reject(new Error())
Falha! O erro é capturado pelo .catch().
.then(dados => ...)
Processa o resultado. Pode retornar outro valor ou outra Promise (encadeamento).
.catch(err => ...)
Captura qualquer erro de qualquer .then() anterior na cadeia.
async.js 
// ── async / await + fetch ──────────────────────
async function buscarPais(nome) {
  try {
    const resposta = await fetch(
      `https://restcountries.com/v3.1/name/${nome}`
    );

    if (!resposta.ok) {
      throw new Error(`Erro HTTP: ${resposta.status}`);
    }

    const [pais] = await resposta.json();  // desestrutura array

    console.log(pais.name.common);       // "Brazil"
    console.log(pais.capital[0]);         // "Brasília"
    console.log(pais.population.toLocaleString());

  } catch (erro) {
    console.error("Deu ruim:", erro.message);
  } finally {
    console.log("Busca concluída.");  // sempre executa
  }
}

buscarPais("brazil");

// ── Timer assíncrono ───────────────────────────
function esperar(ms) {
  return new Promise(resolve => setTimeout(resolve, ms));
}

async function contagem() {
  console.log("3...");
  await esperar(1000);
  console.log("2...");
  await esperar(1000);
  console.log("🚀 Lançar!");
}
contagem();
💡
localStorage: Para salvar dados no navegador sem banco de dados use localStorage.setItem("chave", JSON.stringify(valor)) e JSON.parse(localStorage.getItem("chave")) para recuperar. Os dados persistem mesmo após fechar o navegador.

Exercícios Práticos

A melhor forma de aprender é codar. Cada exercício reforça os conceitos de uma fase. Tente antes de ver a dica!

EX 01 Calculadora com Switch
  • Leia dois números e um operador (+, -, *, /) com prompt().
  • Use switch para calcular e exibir o resultado com alert().
  • Trate a divisão por zero com uma mensagem de erro.
  • Use Number() para converter a string do prompt para número.
💡 Dica: const n1 = Number(prompt("Primeiro número:")); — o prompt sempre retorna string, converta antes de calcular.
EX 02 Analisador de Turma
  • Crie um array de objetos com { nome, nota } para pelo menos 5 alunos.
  • Use .filter() para separar aprovados (nota ≥ 7) de reprovados.
  • Use .map() para criar um array só com os nomes dos aprovados.
  • Use .reduce() para calcular a média geral da turma.
  • Mostre tudo no console com console.log().
💡 Dica: const media = notas.reduce((a, b) => a + b, 0) / notas.length — o segundo argumento do reduce é o valor inicial do acumulador.
EX 03 Lista de Tarefas (DOM)
  • Crie um HTML com um <input>, um <button> e uma <ul>.
  • Ao clicar no botão, crie um <li> com o texto do input usando createElement.
  • Adicione um botão "✕" em cada item para removê-lo da lista.
  • Ao clicar no item, adicione a classe concluido (riscado com CSS).
💡 Dica: crie o botão de deletar dentro da função criarItem(texto) e use li.remove() no click do botão.
EX 04 Buscador de Países
  • Crie um input de texto e um botão "Buscar" no HTML.
  • Ao clicar, use fetch + async/await para buscar de restcountries.com/v3.1/name/NOME.
  • Exiba na página: nome do país, capital, população e bandeira (country.flags.png).
  • Mostre mensagem de erro se o país não for encontrado.
💡 Dica: a API retorna um array, use desestruturação: const [pais] = await res.json();. Verifique res.ok antes do .json().
EX 05 Jogo: Adivinhe o Número
  • Gere um número secreto entre 1 e 100 com Math.floor(Math.random() * 100) + 1.
  • O usuário digita palpites em um input. A cada tentativa, diga "Maior" ou "Menor".
  • Mostre quantas tentativas foram necessárias quando acertar.
  • Adicione um botão "Novo Jogo" que reinicia sem recarregar a página.
💡 Dica: guarde o número secreto e o contador em variáveis no escopo do módulo. Use let para poder reassignar ao reiniciar.
BÔNUS Notas com localStorage
  • Crie um app de bloco de notas: input de título, textarea para o conteúdo, botão "Salvar".
  • Salve as notas em localStorage como array de objetos { id, titulo, texto, data }.
  • Ao carregar a página, recupere e renderize as notas salvas.
  • Adicione botão de excluir em cada nota (remove do array e atualiza localStorage).
💡 Dica: use Date.now() como ID único. Sempre salve o array inteiro: localStorage.setItem("notas", JSON.stringify(notas)).
