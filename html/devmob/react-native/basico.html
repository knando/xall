01 — Ponto de Partida

O que é React Native & Expo Snack?

React Native é um framework criado pelo Meta que permite construir aplicativos reais para Android e iOS usando JavaScript e React. Com o Expo Snack, você escreve e testa no navegador sem instalar nada.

React Native
Framework que transforma componentes JavaScript em elementos nativos reais de iOS e Android.
Expo
Plataforma que simplifica o desenvolvimento RN: ferramentas, bibliotecas e ambiente de build prontos.
Expo Snack
Editor online em snack.expo.dev — escreva e visualize seu app no navegador ou pelo aplicativo Expo Go no celular.
Componente
Bloco reutilizável de UI. Pode ser simples (um botão) ou complexo (uma tela inteira). Sempre retorna JSX.
JSX
Sintaxe parecida com HTML usada dentro do JavaScript para descrever a interface visual.
Props
Parâmetros passados de um componente pai para um filho. São somente leitura dentro do filho.
🚀
Como começar: Acesse snack.expo.dev, clique em New Snack e você já tem um app funcional. Edite o App.js e veja as mudanças em tempo real no emulador ao lado.

Estrutura mínima de um componente

App.js 
// 1️⃣  Importações — sempre no topo
import React from 'react';
import { View, Text, StyleSheet } from 'react-native';

// 2️⃣  Definição do componente — função que retorna JSX
export default function App() {
  return (
    <View style={styles.container}>
      <Text style={styles.titulo}>Olá, mundo!</Text>
      <Text>Meu primeiro app React Native 🚀</Text>
    </View>
  );
}

// 3️⃣  Estilos — equivalente ao CSS, mas em JavaScript
const styles = StyleSheet.create({
  container: {
    flex: 1,
    alignItems: 'center',
    justifyContent: 'center',
    backgroundColor: '#f0f4ff',
  },
  titulo: {
    fontSize: 24,
    fontWeight: 'bold',
    marginBottom: 8,
  },
});
⚠️
Diferença do HTML: Em React Native não existe <div>, <p> ou <button>. No lugar, usamos <View>, <Text> e <TouchableOpacity>. Todo texto obrigatoriamente deve estar dentro de um <Text>.

03 — Dados que Mudam

Estado & Hooks

Estado (state) é qualquer dado que pode mudar durante a vida do componente e que, quando muda, faz a tela se atualizar automaticamente. Os Hooks são funções especiais do React que gerenciam esse estado.

useState
Cria uma variável de estado. Retorna [valor, setValor]. Quando setValor é chamado, o componente re-renderiza.
useEffect
Executa um efeito colateral (buscar dados, timer) após a renderização. Controla quando rodar pelo array de dependências.
useCallback
Memoriza uma função para não recriar a cada render, evitando re-renders desnecessários em componentes filhos.
useContext
Lê um contexto global — útil para passar dados (ex: usuário logado) sem "prop drilling" por vários níveis.

useState — o hook mais importante

Contador.js 
import React, { useState } from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';

export default function Contador() {
  // useState retorna [valorAtual, funcaoQueAtualiza]
  const [contagem, setContagem] = useState(0);   // valor inicial: 0
  const [nome, setNome] = useState('');          // valor inicial: string vazia

  return (
    <View style={s.container}>

      {/* Exibe o estado atual */}
      <Text style={s.numero}>{contagem}</Text>

      {/* Botões que ATUALIZAM o estado */}
      <View style={s.row}>
        <TouchableOpacity style={s.btn}
          onPress={() => setContagem(c => c - 1)}>
          <Text style={s.btnTxt}>−</Text>
        </TouchableOpacity>
        <TouchableOpacity style={[s.btn, s.btnGreen]}
          onPress={() => setContagem(c => c + 1)}>
          <Text style={s.btnTxt}>+</Text>
        </TouchableOpacity>
      </View>

    </View>
  );
}

const s = StyleSheet.create({
  container: { flex: 1, alignItems: 'center', justifyContent: 'center' },
  numero:    { fontSize: 72, fontWeight: 'bold', color: '#6366f1', marginBottom: 32 },
  row:       { flexDirection: 'row', gap: 16 },
  btn:       { backgroundColor: '#ef4444', width: 60, height: 60,
               borderRadius: 30, alignItems: 'center', justifyContent: 'center' },
  btnGreen:  { backgroundColor: '#22c55e' },
  btnTxt:    { color: '#fff', fontSize: 28, fontWeight: 'bold' },
});
💡
Regra de ouro: Nunca modifique o estado diretamente (contagem = contagem + 1). Sempre use a função setter (setContagem(c => c + 1)), caso contrário a tela não atualiza.

Configurando o Stack Navigator

App.js — configuração de navegação 
import React from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';

import HomeScreen     from './screens/HomeScreen';
import DetalhesScreen from './screens/DetalhesScreen';

// Cria o objeto Stack com as telas disponíveis
const Stack = createNativeStackNavigator();

export default function App() {
  return (
    {/* NavigationContainer SEMPRE envolve tudo */}
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Home">

        {/* Cada Screen = uma tela acessível */}
        <Stack.Screen
          name="Home"
          component={HomeScreen}
          options={{ title: 'Minhas Tarefas' }}
        />

        <Stack.Screen
          name="Detalhes"
          component={DetalhesScreen}
          options={{ title: 'Editar Tarefa' }}
        />

      </Stack.Navigator>
    </NavigationContainer>
  );
}

Método O que faz Exemplo
navigate(name, params) Vai para uma tela, passando dados opcionais navigate('Detalhes', { id: 1 })
goBack() Volta para a tela anterior da pilha navigation.goBack()
replace(name) Substitui a tela atual (sem acumular histórico) navigation.replace('Login')
reset(state) Reseta toda a pilha de navegação navigation.reset({ index: 0, routes: [{ name: 'Home' }] })
05 — Projeto Completo

App CRUD com Duas Telas

Vamos construir um gerenciador de tarefas completo: criar, listar, editar e deletar itens — com dados fluindo entre a tela Home e a tela de Detalhes.

🗂️
Estrutura de arquivos no Snack:
App.js — configura a navegação
screens/HomeScreen.js — lista + criação (C e R)
screens/DetalhesScreen.js — edição + deleção (U e D)

App.js — Raiz da navegação + estado global de tarefas

App.js 
import React, { useState } from 'react';
import { NavigationContainer } from '@react-navigation/native';
import { createNativeStackNavigator } from '@react-navigation/native-stack';
import HomeScreen     from './screens/HomeScreen';
import DetalhesScreen from './screens/DetalhesScreen';

const Stack = createNativeStackNavigator();

// Lista inicial de tarefas — serve de banco de dados em memória
const INICIAIS = [
  { id: '1', titulo: 'Estudar React Native', feito: false },
  { id: '2', titulo: 'Publicar app na Play Store', feito: false },
];

export default function App() {
  // Estado centralizado no App — compartilhado pelas duas telas via props
  const [tarefas, setTarefas] = useState(INICIAIS);

  // CREATE — adiciona uma nova tarefa
  const adicionarTarefa = (titulo) => {
    const nova = { id: Date.now().toString(), titulo, feito: false };
    setTarefas(prev => [...prev, nova]);
  };

  // UPDATE — atualiza o título de uma tarefa existente
  const atualizarTarefa = (id, novoTitulo) => {
    setTarefas(prev =>
      prev.map(t => t.id === id ? {...t, titulo: novoTitulo} : t)
    );
  };

  // DELETE — remove uma tarefa pelo id
  const deletarTarefa = (id) => {
    setTarefas(prev => prev.filter(t => t.id !== id));
  };

  return (
    <NavigationContainer>
      <Stack.Navigator initialRouteName="Home"
        screenOptions={{
          headerStyle: { backgroundColor: '#6366f1' },
          headerTintColor: '#fff',
          headerTitleStyle: { fontWeight: 'bold' },
        }}
      >
        <Stack.Screen name="Home" options={{ title: '📋 Minhas Tarefas' }}>
          {/* Passamos as funções como props via children render */}
          {(props) =>
            <HomeScreen
              {...props}
              tarefas={tarefas}
              onAdicionar={adicionarTarefa}
            />
          }
        </Stack.Screen>

        <Stack.Screen name="Detalhes" options={{ title: '✏️ Editar Tarefa' }}>
          {(props) =>
            <DetalhesScreen
              {...props}
              onAtualizar={atualizarTarefa}
              onDeletar={deletarTarefa}
            />
          }
        </Stack.Screen>
      </Stack.Navigator>
    </NavigationContainer>
  );
}

screens/HomeScreen.js — Listar (READ) e Criar (CREATE)

screens/HomeScreen.js 
import React, { useState } from 'react';
import {
  View, Text, TextInput, TouchableOpacity,
  FlatList, StyleSheet, Alert
} from 'react-native';

export default function HomeScreen({ navigation, tarefas, onAdicionar }) {
  const [texto, setTexto] = useState('');

  // CREATE — chama a função do App.js
  const handleAdicionar = () => {
    if (texto.trim() === '') {
      Alert.alert('Atenção', 'Digite um título para a tarefa.');
      return;
    }
    onAdicionar(texto.trim());
    setTexto('');   // limpa o campo após adicionar
  };

  // Renderiza cada item da lista — navega ao clicar
  const renderItem = ({ item }) => (
    <TouchableOpacity
      style={s.item}
      onPress={() => navigation.navigate('Detalhes', {
        tarefaId: item.id,
        titulo:   item.titulo,
      })}
    >
      <Text style={s.itemTxt}>{item.titulo}</Text>
      <Text style={s.seta}>›</Text>
    </TouchableOpacity>
  );

  return (
    <View style={s.container}>

      {/* Campo para nova tarefa */}
      <View style={s.row}>
        <TextInput
          style={s.input}
          value={texto}
          onChangeText={setTexto}
          placeholder="Nova tarefa..."
          returnKeyType="done"
          onSubmitEditing={handleAdicionar}
        />
        <TouchableOpacity style={s.btnAdd} onPress={handleAdicionar}>
          <Text style={s.btnAddTxt}>+</Text>
        </TouchableOpacity>
      </View>

      {/* READ — lista todas as tarefas */}
      <FlatList
        data={tarefas}
        keyExtractor={(item) => item.id}
        renderItem={renderItem}
        ListEmptyComponent={
          <Text style={s.vazio}>Nenhuma tarefa ainda. Adicione uma! 🎯</Text>
        }
      />

    </View>
  );
}

screens/DetalhesScreen.js — Editar (UPDATE) e Deletar (DELETE)

screens/DetalhesScreen.js 
import React, { useState } from 'react';
import {
  View, Text, TextInput, TouchableOpacity,
  StyleSheet, Alert
} from 'react-native';

export default function DetalhesScreen({ route, navigation, onAtualizar, onDeletar }) {
  // Recebe os dados enviados pela HomeScreen via route.params
  const { tarefaId, titulo } = route.params;

  // Estado local inicializado com o título recebido
  const [novoTitulo, setNovoTitulo] = useState(titulo);

  // UPDATE — salva e volta para a Home
  const handleSalvar = () => {
    if (novoTitulo.trim() === '') {
      Alert.alert('Atenção', 'O título não pode ficar vazio.');
      return;
    }
    onAtualizar(tarefaId, novoTitulo.trim());  // atualiza no App.js
    navigation.goBack();                       // volta para a Home
  };

  // DELETE — pede confirmação antes de deletar
  const handleDeletar = () => {
    Alert.alert(
      'Confirmar exclusão',
      `Deletar "${titulo}"?`,
      [
        { text: 'Cancelar', style: 'cancel' },
        {
          text: 'Deletar', style: 'destructive',
          onPress: () => {
            onDeletar(tarefaId);    // remove no App.js
            navigation.goBack();    // volta para a Home
          },
        },
      ]
    );
  };

  return (
    <View style={s.container}>
      <Text style={s.label}>Título da tarefa</Text>
      <TextInput
        style={s.input}
        value={novoTitulo}
        onChangeText={setNovoTitulo}
        autoFocus
      />

      {/* Botão de salvar — UPDATE */}
      <TouchableOpacity style={s.btnSalvar} onPress={handleSalvar}>
        <Text style={s.btnTxt}>💾 Salvar alterações</Text>
      </TouchableOpacity>

      {/* Botão de deletar — DELETE */}
      <TouchableOpacity style={s.btnDeletar} onPress={handleDeletar}>
        <Text style={s.btnTxt}>🗑️ Deletar tarefa</Text>
      </TouchableOpacity>
    </View>
  );
}

const s = StyleSheet.create({
  container:  { flex: 1, padding: 24, backgroundColor: '#f8f9fe' },
  label:      { fontSize: 13, fontWeight: '600', color: '#6366f1',
               marginBottom: 8, textTransform: 'uppercase', letterSpacing: 1 },
  input:      { backgroundColor: '#fff', borderWidth: 1, borderColor: '#d1d5db',
               borderRadius: 10, padding: 14, fontSize: 16, marginBottom: 24 },
  btnSalvar:  { backgroundColor: '#6366f1', padding: 16, borderRadius: 10,
               alignItems: 'center', marginBottom: 12 },
  btnDeletar: { backgroundColor: '#ef4444', padding: 16, borderRadius: 10,
               alignItems: 'center' },
  btnTxt:     { color: '#fff', fontWeight: 'bold', fontSize: 16 },
});
Operação Tela Função no estado Resultado
CREATE HomeScreen setTarefas(prev => [...prev, nova]) Nova tarefa aparece na lista
READ HomeScreen FlatList data={tarefas} Lista renderizada automaticamente
UPDATE DetalhesScreen prev.map(t => t.id === id ? {...t, titulo} : t) Item atualizado sem recriar lista
DELETE DetalhesScreen prev.filter(t => t.id !== id) Item removido da lista
✅
Padrão de dados imutáveis: Nunca modifique o array diretamente (tarefas.push(...)). Sempre crie uma cópia nova com map, filter ou spread ([...prev, novo]). Isso garante que o React detecte a mudança e re-renderize a tela.

Exercícios

EX 01 Contador com Histórico
  • Crie uma tela com um número central, botões + e −.
  • Use useState para o valor do contador.
  • Adicione um segundo useState com um array que guarda cada alteração feita.
  • Exiba o histórico abaixo do contador com um FlatList.
💡 Dica: ao incrementar, faça setHistorico(prev => [...prev, { acao: '+1', valor: novoVal }])
EX 02 Perfil com Duas Telas
  • Crie duas telas: ListaScreen (lista de usuários) e PerfilScreen.
  • Na ListaScreen, mostre 3–5 nomes em uma FlatList.
  • Ao tocar num nome, navegue para PerfilScreen passando o nome e a idade.
  • Na PerfilScreen, exiba o nome, a idade e um botão "Voltar".
💡 Dica: use navigation.navigate('Perfil', { nome, idade }) e recupere com route.params.
EX 03 Lista de Compras com Checkbox
  • Monte uma lista de compras com itens que podem ser marcados como comprados.
  • Cada item tem nome e comprado: false no estado inicial.
  • Ao tocar no item, alterne o campo comprado usando map.
  • Itens comprados devem aparecer com texto riscado (textDecorationLine: 'line-through').
💡 Dica: setItens(prev => prev.map(i => i.id === id ? {...i, comprado: !i.comprado} : i))
EX 04 Busca em Tempo Real
  • Crie uma lista de pelo menos 10 cidades brasileiras.
  • Adicione um TextInput de busca no topo da tela.
  • Use filter para mostrar apenas as cidades que contêm o texto digitado.
  • A filtragem deve acontecer a cada letra digitada, em tempo real.
💡 Dica: filtre diretamente na prop data do FlatList: data={cidades.filter(c => c.toLowerCase().includes(busca.toLowerCase()))}
EX 05 CRUD de Notas com 3 Telas
  • Expanda o exemplo do tutorial adicionando uma terceira tela: NovaNota.
  • Na HomeScreen, adicione um botão "+" que navega para NovaNota.
  • NovaNota tem campos de título e conteúdo, e um botão "Salvar".
  • Ao salvar, a nota volta para a Home e aparece na lista.
  • Ao tocar numa nota, vai para Detalhes com edição e deleção.
💡 Dica: crie a função adicionarNota(titulo, conteudo) no App.js e passe via prop para NovaNota usando o padrão children render do tutorial.

Gabarito parcial — Exercício 03 (lista de compras)

ListaCompras.js — trecho principal 
const [itens, setItens] = useState([
  { id: '1', nome: 'Arroz',    comprado: false },
  { id: '2', nome: 'Feijão',  comprado: false },
  { id: '3', nome: 'Café',    comprado: false },
]);

// Alterna comprado/não-comprado sem mutar o estado original
const toggleItem = (id) =>
  setItens(prev =>
    prev.map(i => i.id === id ? {...i, comprado: !i.comprado} : i)
  );

// Renderiza o item com estilo condicional
const renderItem = ({ item }) => (
  <TouchableOpacity
    style={s.item}
    onPress={() => toggleItem(item.id)}
  >
    {/* Ícone muda conforme estado */}
    <Text style={s.icon}>{item.comprado ? '✅' : '⬜'}</Text>
    <Text style={[
      s.nome,
      item.comprado && s.riscado   {/* aplica estilo extra se comprado */}
    ]}>{item.nome}</Text>
  </TouchableOpacity>
);

// Nos estilos:
riscado: { textDecorationLine: 'line-through', color: '#9ca3af' },
